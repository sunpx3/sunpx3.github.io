Java重读IO流

2016-11-26

以前学习IO的时候看到一堆的类与api,没有相应的知识结构，经常一脸懵逼，所以现在重新梳理下。
1、什么是IO流？
IO流是存储设备之间的一种数据交换，形象的称作流。
2、流的分类
按方向分: 输入输出流
按流的数据单位: 字符流与字节流，顶屋基类为Reader/Write, InputStream/OutPutStream
3、为什么一个读写操作需要这么多类？
当数据被写入IO后，已经被序列化了，是以byte为单位的，而FileInputStream 与FileOutPutStream只提供了简易的read(),write()方法
只能一个byte一个byte的读取，所以为了开发方便，出现了各种“辅助”的类，用来批量存读取。

mybatis生成实体类配置文件

2016-11-10

MyBatis 真正的力量是在映射语句中。这里是奇迹发生的地方。对于所有的力量,SQL 映射的 XML 文件是相当的简单。当然如果你将它们和对等功能的 JDBC 代码来比较,你会发现映射文件节省了大约 95%的代码量。MyBatis 的构建就是聚焦于 SQL 的,使其远离于普通的方式。
SQL 映射文件有很少的几个顶级元素(按照它们应该被定义的顺序):
● cache – 配置给定命名空间的缓存。
● cache-ref – 从其他命名空间引用缓存配置。
● resultMap – 最复杂,也是最有力量的元素,用来描述如何从数据库结果集中来加载你的对象。
● parameterMap – 已经被废弃了!老式风格的参数映射。内联参数是首选,这个元素可能在将来被移除。这里不会记录。
● sql – 可以重用的 SQL 块,也可以被其他语句引用。
● insert – 映射插入语句
● update – 映射更新语句
● delete – 映射删除语句
● select – 映射查询语句
下一部分将从语句本身开始来描述每个元素的细节。
select
